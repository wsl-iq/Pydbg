# Contributing to Pydbg

Thank you for your interest in contributing to Pydbg.

## Getting Started

1. Fork the repository.
2. Clone your fork.
3. Create a new branch.
4. Make your changes.
5. Test your changes.
6. Commit your work.
7. Open a Pull Request.

## Code Style

- Follow Python best practices.
- Write clean and readable code.
- Keep commits focused on a single change.

## Reporting Bugs

Please provide:

- Operating System
- Python Version
- Pydbg Version
- Steps to reproduce
- Expected behavior
- Actual behavior

## Feature Requests

Before opening a feature request, please check existing issues.

Describe:

- The problem.
- Your proposed solution.
- Why it would benefit the project.
